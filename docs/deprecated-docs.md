# Deprecated and Historical Documents

This project has older proposal documents, investigation notes, and flow descriptions that may no longer match the current Web implementation.

These documents are not deleted because they may contain useful context, assets, or design reasoning. They should not be treated as current instructions when they conflict with the canonical docs.

## Canonical Docs

Use these first:

- `README.md`
- `PLAN.md`
- `docs/product-spec.md`
- `docs/work-plan.md`
- `docs/rules/cucumber5.md`
- `AGENTS.md`
- `AI_RULES.md`
- `WORKFLOW.md`

## Historical Reference Only

The following categories are historical unless a future phase explicitly updates them:

- `.docx` proposal files under `docs/`
- old investigation notes such as UI investigation or home UI investigation files
- docs that mention old routes like `/play/cucumber5`, `/lobby/cucumber5`, `/auth/login`, `/stats`, or `/settings`
- docs that describe 5-digit room codes
- docs that describe the old 60-card rule

## Rule for Future Work

If a historical document contains a useful idea, copy the idea into a current canonical doc or implementation issue before acting on it. Do not silently treat historical notes as current requirements.
